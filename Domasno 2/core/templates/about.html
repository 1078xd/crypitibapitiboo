About Crypitibapitiboo

Crypitibapitiboo is a web-based system designed to collect, process, and analyze historical cryptocurrency market data. The goal of the project is to transform raw exchange data into meaningful insights through structured pipelines and technical analysis.

The application is built entirely in Python, chosen for its strong ecosystem of data-processing libraries and excellent support for API interaction and automation. Market data is collected and stored in a structured format that can be easily extended to relational databases such as PostgreSQL.

Why Binance?

Multiple data sources were evaluated during development, including blockchain-level data, web scraping, decentralized exchanges, and public market-data aggregators. Each option had significant limitations such as complexity, instability, limited historical depth, or restrictive pricing models.

After careful evaluation, the Binance public API was selected as the primary data source. Binance provides stable, well-documented endpoints, long historical OHLCV data, and supports hundreds of actively traded USDT pairs â€” all without requiring authentication.

Architecture & Data Pipeline

Crypitibapitiboo follows a Pipe-and-Filter architecture. Instead of a monolithic process, the system is divided into independent filters, each responsible for a single task:

  • Discovering available trading pairs
  • Defining historical time ranges
  • Parallel data extraction using multithreading
  • Cleaning and formatting OHLCV records
  • Storing results in a unified dataset

What This Project Delivers

The result is an automated, scalable platform capable of producing up-to-date cryptocurrency datasets and performing multi-timeframe technical analysis. Beyond the data itself, the project demonstrates practical experience with data engineering patterns, API-driven systems, and the foundations of quantitative market analysis.
